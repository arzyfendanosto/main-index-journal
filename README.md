# main-index-journal

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名、网站或项目，主要作为静态页面集合的存储与展示空间。所有页面均为独立制作，内容涵盖不同主题，便于统一管理和访问。

## 目录说明

```
main-index-journal/
├── index.html          # 仓库主页索引
├── pages/              # 存放所有独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源（样式、图片、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：仓库主入口，列出所有已归档页面的链接。
- `pages/`：每个 HTML 页面作为独立文件存放，文件名反映内容主题。
- `assets/`：存放页面共用的样式表、JavaScript 脚本和图片资源。

## 页面归档说明

仓库中收录的 HTML 页面均为独立制作，内容保持原始状态。可能包含以下类型：

- 静态信息展示页
- 简单工具或交互页面
- 实验性或练习性质页面
- 其他临时归档页面

每个页面均可通过 `pages/` 目录下的对应文件直接访问。页面之间无依赖关系，可单独打开使用。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并在 `index.html` 中添加对应链接。
- 更新页面：直接修改对应文件，注意保持文件路径不变。
- 移除页面：删除文件并从 `index.html` 中移除链接。
- 资源管理：公共资源建议放入 `assets/` 目录，避免重复。

本仓库不设自动部署，更新后需手动推送至 GitHub Pages 或其他托管服务以生效。
